# Coin项目 Postman 接口文档使用说明

## 📋 文件说明

本目录包含 Coin 项目 API 的完整 Postman 集合配置：

- **`postman_collection.json`** - Postman 集合文件，包含所有 API 接口
- **`postman_environment.json`** - 环境变量配置文件
- **`POSTMAN_README.md`** - 本使用说明文档

## 🚀 快速开始

### 1. 导入集合

1. 打开 Postman 应用
2. 点击 **Import** 按钮
3. 选择 **File** 标签页
4. 上传 `postman_collection.json` 文件
5. 重复步骤上传 `postman_environment.json` 文件作为环境

### 2. 切换环境

在 Postman 右上角的下拉菜单中选择 **"Coin API 环境"**

### 3. 首次使用

1. 先执行 **"1. 认证管理模块"** → **"用户登录"** 接口
2. 使用测试账号：`demo` / `demo12345`
3. 登录成功后，Token 会自动保存到环境变量中

## 🔧 环境变量说明

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `baseUrl` | API 基础 URL | `https://api.example.com` |
| `token` | 用户认证 Token | 登录后自动设置 |
| `username` | 当前登录用户名 | 登录后自动设置 |
| `userId` | 用户 ID | 登录后自动设置 |
| `airdropId` | 空投活动 ID | `1` |
| `powerId` | 算力产品 ID | `1` |
| `orderId` | 订单 ID | 购买接口返回后设置 |
| `taskId` | 任务 ID | `1` |
| `withdrawalId` | 提现 ID | 提现接口返回后设置 |
| `messageId` | 消息 ID | `1` |
| `symbol` | 交易对符号 | `BTC/USDT` |
| `testUsername` | 测试用户名 | `demo` |
| `testPassword` | 测试用户密码 | `demo12345` |
| `resetToken` | 密码重置 Token | 忘记密码流程中设置 |

## ⚠️ 注意事项

1. **认证要求**：除注册和登录接口外，其他接口都需要有效的 Token
2. **测试环境**：使用测试环境 URL，避免影响生产数据
3. **数据清理**：测试完成后请清理测试数据
4. **并发限制**：注意 API 的速率限制，避免过于频繁的请求

## 🐛 常见问题

### Q: 登录接口返回成功但没有保存 Token？
A: 检查登录接口的 Test 脚本是否正确执行，确保响应格式符合预期。

### Q: 其他接口返回 401 错误？
A: 确认 Token 是否已正确设置且未过期，尝试重新登录。

### Q: 文件上传接口失败？
A: 确保文件大小和格式符合要求，检查请求头是否正确设置。

### Q: 环境变量没有更新？
A: 确认已选择正确的环境，检查脚本是否有权限修改变量。
